Configuration History Page Feature, Developed at Entrust during my Internship

On an Entrust webpage I created a configuration history page which showed recent configuration settings which had been updated or created. There was a drop down at the top of the page allowing for a time selection which specified how many hours to look back on. This was helpful for situations when something broke because of configuration changes. This tool is a much more readable version of log files. This required me to build a backend call and create a new webpage styling the frontend to conform to the other page's style.

Time Zone Choice Feature in Log and Audit Pages, Developed at Entrust during my Internship

On an Entrust webpage I added a configuration setting which describes time zones one can choose from. Whichever time zone is chosen dictates how time is displayed in logging and audit descriptions. This required interaction with moment.js and moment-timezone.js libraries. This project allows teams working in different time zones to have clarification for what time zone they are looking at when production, development and the database are taking place across timezones.
